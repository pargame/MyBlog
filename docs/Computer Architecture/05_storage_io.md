# 보조기억장치와 입출력장치


이 문서에서는 보조기억장치(RAID 포함)와 입출력 기법을 간단히 정리합니다.

## __RAID
- RAID의 목적: 신뢰성(복제)과 성능(스트라이프)을 제공
- 주요 레벨
  - RAID 0: 스트라이프(성능 향상, 무결성 없음)
  - RAID 1: 미러(복제, 읽기 성능 증가, 쓰기 오버헤드)
  - RAID 5/6: 패리티 기반 보호(스토리지 효율과 복원력 균형)
- 소프트웨어 RAID vs 하드웨어 RAID: 관리 편의성과 성능/장애 회복 측면의 차이

## __입출력 기법
- 폴링(polling) vs 인터럽트 기반 I/O
  - 폴링: CPU가 장치 상태를 주기적으로 확인 — 단순하지만 비효율적
  - 인터럽트: 장치가 CPU에 알림 — 효율적이나 컨텍스트 스위칭 비용 존재
- DMA(Direct Memory Access)
  - CPU 개입을 최소화하고 장치가 메모리에 직접 데이터를 전송하도록 허용
  - 대량 데이터 전송에 유리
- I/O 버퍼링과 캐싱: 스루풋과 지연(latency)을 조절하기 위해 사용

입출력 설계는 전체 시스템 성능과 응답성에 큰 영향을 미칩니다.
