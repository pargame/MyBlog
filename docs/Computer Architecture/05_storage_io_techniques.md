# 입출력 기법 심화 (I/O Techniques Deep Dive)

이 문서는 다양한 I/O 처리 방식과 성능/자원 사용 관점에서의 설계 고려사항을 설명합니다.

## 폴링 vs 인터럽트
- 폴링(Polling): CPU가 장치 상태를 반복적으로 확인
  - 장점: 구현 단순
  - 단점: CPU 소비가 크고 비효율적
- 인터럽트(Interrupt): 장치가 이벤트 발생 시 CPU에 알림
  - 장점: 효율적, 대기 없이 이벤트 처리 가능
  - 단점: 컨텍스트 스위칭 비용, 우선순위 관리 필요

## DMA (Direct Memory Access)
- 장치가 CPU 개입 없이 메모리에 직접 데이터 전송
- 대량 데이터 전송에 유리하며 CPU 부하를 감소
- DMA 엔진과 메모리 일관성 고려 필요

## 비동기 I/O와 폴리/프레임워크
- 비동기 I/O: 작업을 시작하고 완료 시 콜백/이벤트로 알림
- 이벤트 루프(예: epoll/kqueue)와 비동기 라이브러리 사용으로 높은 동시성 확보

## 버퍼링과 캐싱
- I/O 버퍼링: 데이터 전송 단위를 묶어 효율 극대화
- 쓰기 버퍼와 쓰기 백(behind) 전략: 데이터 안전성 vs 성능 트레이드오프

## 성능 측정 및 튜닝
- 측정 지표: IOPS, throughput, latency
- 병목 분석: CPU, 버스 대역, 스토리지 지연, 동시성 제약

## 사례
- 고성능 네트워크 스택: zero-copy, kernel bypass 기술
- 대용량 데이터 처리: 배치 전송 + 압축, SSD 최적화
