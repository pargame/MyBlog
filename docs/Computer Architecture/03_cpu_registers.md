# 레지스터 (Registers)

레지스터는 CPU 내부의 소형 고속 저장소입니다. 주로 프로그램 카운터(PC), 상태레지스터(flags), 일반 목적 레지스터(general-purpose registers)로 구성됩니다.

## 역할
- 연산의 피연산자와 결과를 저장하여 메모리 접근을 줄임
- 제어 흐름을 관리(PC)
- 상태/플래그를 통해 조건 분기와 예외 처리를 지원

## 종류
- 일반 목적 레지스터: 연산에 직접 쓰이는 레지스터들(R0..Rn)
- 특수 목적 레지스터: PC, SP(스택 포인터), LR(링크 레지스터), 상태 레지스터
- 레지스터 창/window: 일부 아키텍처(예: SPARC)에서 함수 호출을 빠르게 처리하기 위해 사용

## 설계 고려사항
- 레지스터 수: 많은 레지스터는 컴파일러 최적화에 유리하지만 하드웨어 복잡도 증가
- 읽기/쓰기 포트: 동시 접근을 위한 포트 수는 설계 복잡도와 면적에 영향
- 물리/논리 레지스터 맵핑: 아웃 오브 오더나 레지스터 리네이밍을 위해 필요

## 구현 팁
- 레지스터 파일은 빠른 접근을 위해 SRAM으로 구현되는 경우가 많음
- 레지스터 리네이밍은 엑세스 충돌을 줄이고 파이프라인 성능을 높임

### Register file sketch
```
	+---------------------------+
	| R0 | R1 | R2 | R3 | ...  |
	+---------------------------+
	 ^    ^    ^    ^
	 |    |    |    |---> used as operands
	 |    |    |---> temporary
	 |    |---> result
	 |---> frame pointer / base
```

예시: 간단한 레지스터 사용(의사어셈블리)
```asm
LOAD R1, [R0]  ; R0 holds address
ADD  R2, R1, #4
STORE R2, [R0]
```
