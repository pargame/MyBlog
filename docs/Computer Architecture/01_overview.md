# 구조 개요


이 문서는 컴퓨터 구조의 핵심 개념과 주요 부품의 역할, 그리고 전체 시스템이 어떻게 연결되는지 한눈에 보여줍니다. 실무에서 자주 마주치는 병목, 설계 포인트, 그리고 구조적 시각화 방법도 함께 제시합니다.


## 정보와 데이터 표현
- 컴퓨터는 모든 정보를 0과 1(비트)로 표현합니다.
- 숫자, 문자, 명령어 등은 다양한 인코딩 방식(정수, 부동소수점, 문자코드 등)으로 저장됩니다.


## 주요 부품 요약
- **CPU**: 연산과 제어의 중심, 프로그램 실행 담당
- **메모리**: 작업 데이터와 명령어 임시 저장(RAM)
- **스토리지**: 대용량 영구 저장(SSD/HDD)
- **I/O**: 외부와의 데이터 교환
- **버스/인터커넥트**: 부품 간 데이터 전달


## 구조적 시각화와 설계 관점
- 계층적 구조: ISA → 마이크로아키텍처 → 회로 구현
- 데이터 흐름: 입력 → 처리 → 출력
- 성능 관점: 병렬성, 캐시, 메모리 계층, 병목 포인트

### 연습 문제
1) 컴퓨터 구조에서 병목이 발생하는 대표적인 지점 2가지를 들고, 각각이 전체 성능에 미치는 영향을 간단히 설명하라.

### 예시: 간단한 컴퓨터 구조 지도
- 노드: [CPU] — [L1 Cache] — [L2 Cache] — [DRAM]
- 엣지 타입: solid line = 데이터 흐름, dashed line = 제어/인터럽트
- 강조: CPU에서 DRAM으로 가는 경로에 캐시 미스가 잦으면 해당 경로를 굵게 표시

이 지도는 `graph.html`의 지식 그래프(노드/링크)와 비슷한 방식으로 문서 간 연관성을 표현할 수 있습니다. 각 문서에서 `[[Wikilink]]` 또는 상대경로 링크를 사용해 상호 연결하면 그래프 빌더가 자동으로 엣지를 생성합니다.
