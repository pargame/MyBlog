# 1. 컴퓨터 구조의 큰 그림

이 문서는 컴퓨터 구조의 전체적인 개념을 한눈에 정리합니다. 설계 관점에서 컴퓨터가 어떻게 정보를 이해하고 처리하는지, 주요 구성 요소가 무엇인지, 그리고 이들을 연결해 "컴퓨터 구조 지도"를 그리는 방법을 다룹니다.

## __컴퓨터가 이해하는 정보
- 컴퓨터는 모든 정보를 이진(0과 1)으로 표현합니다. 숫자, 문자, 명령어 등 다양한 데이터가 비트의 조합으로 인코딩됩니다.
- 데이터 표현 방식(정수, 부동소수점, 문자 인코딩 등)은 연산과 저장에 직접적인 영향을 줍니다.

## __컴퓨터의 핵심 부품
- CPU: 명령어를 해석하고 실행하는 중앙 처리 장치
- 메모리(RAM): 작업 중인 데이터와 명령어를 임시 저장
- 보조기억장치: 대용량 영구 저장(SSD/HDD)
- 입출력 장치(I/O): 사용자 및 외부 장치와의 인터페이스
- 버스와 인터커넥트: 부품 간 데이터 전송을 담당

## __컴퓨터 구조 지도 그리기
- 계층적 관점: ISA(Instruction Set Architecture) → 마이크로아키텍처 → 구현(회로)
- 데이터 흐름 관점: 입력 → 처리(연산/메모리/제어) → 출력
- 성능 관점: 병렬성(파이프라인, 슈퍼스칼라), 캐시 구조, 메모리 계층

### 지도 그리기: 권장 프로세스
1. 엔티티 식별: CPU, 레지스터, 캐시, 메모리, 버스, I/O 등 주요 엔티티를 노드로 정의합니다.
2. 관계 정의: 데이터 흐름(읽기/쓰기), 제어(인터럽트/신호), 물리적 연결(버스, 채널) 등 엣지로 표현합니다.
3. 레벨(계층) 지정: ISA·마이크로아키텍처·물리 계층으로 노드를 색상/레이어로 구분합니다.
4. 강조 포인트: 성능 분석을 위해 핫스팟(병목), 병렬성 지점(파이프라인 단계), 캐시 미스 경로 등을 시각적으로 강조합니다.

### 예시: 간단한 컴퓨터 구조 지도
- 노드: [CPU] — [L1 Cache] — [L2 Cache] — [DRAM]
- 엣지 타입: solid line = 데이터 흐름, dashed line = 제어/인터럽트
- 강조: CPU에서 DRAM으로 가는 경로에 캐시 미스가 잦으면 해당 경로를 굵게 표시

이 지도는 `graph.html`의 지식 그래프(노드/링크)와 비슷한 방식으로 문서 간 연관성을 표현할 수 있습니다. 각 문서에서 `[[Wikilink]]` 또는 상대경로 링크를 사용해 상호 연결하면 그래프 빌더가 자동으로 엣지를 생성합니다.
