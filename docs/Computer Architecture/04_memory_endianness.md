# 메모리에 바이트를 밀어 넣는 순서 — 빅 엔디안과 리틀 엔디안

엔디안(Endianness)은 다중 바이트 값이 메모리에 저장되는 바이트 순서를 정의합니다. 시스템 간 데이터 교환과 이식성에 직접적인 영향을 미칩니다.

## 빅 엔디안 (Big Endian)
- 설명: 상위 바이트를 낮은 메모리 주소에 저장
- 장점: 사람이 읽는 순서와 메모리 표현이 유사해 디버깅이 용이
- 예: 네트워크 바이트 오더

## 리틀 엔디안 (Little Endian)
- 설명: 하위 바이트를 낮은 메모리 주소에 저장
- 장점: 가변 길이 연산에서 낮은 바이트 접근이 용이(예: 덧셈 시 carry 전파)
- 예: x86 아키텍처

## 교환 및 변환
- 이식성 고려: 파일 포맷, 네트워크 프로토콜 설계 시 명시적 바이트 오더 정의 필요
- 코드 예시: 호스트-네트워크 바이트 오더 변환(htonl/ntohl)

## 혼합 엔디안과 실무 팁
- 일부 시스템은 혼합 엔디안을 사용하기도 함 — 주의가 필요
- 메모리 뷰를 조작하는 경우(캐스팅, 포인터 연산) 바이트 오더를 항상 고려
