# RAID 상세

이 문서는 다양한 RAID 레벨의 동작, 성능/신뢰성 특성, 그리고 장애 복구 시나리오를 다룹니다.

## RAID 레벨 요약
- RAID 0 (Striping)
  - 데이터 스트라이핑을 통해 성능 향상
  - 장애 허용 불가(디스크 하나라도 손상되면 데이터 손실)
- RAID 1 (Mirroring)
  - 미러링으로 데이터 복제, 높은 읽기 신뢰성
  - 저장 효율 50%
- RAID 5 (Striping with parity)
  - 블록 단위 패리티 분산으로 복구 가능
  - 단일 디스크 장애 복구 가능, 쓰기 성능 오버헤드 존재
- RAID 6 (Dual parity)
  - 두 개의 패리티 블록으로 이중 디스크 장애 허용

## 성능 및 복원
- 읽기 성능: 스트라이핑으로 병렬 읽기 가능 → 대역폭 향상
- 쓰기 성능: 패리티 계산으로 인한 쓰기 오버헤드
- 복원 시 고려사항: rebuild 동안의 성능 저하와 추가 장애 위험

## 구현 옵션
- 하드웨어 RAID vs 소프트웨어 RAID
  - 하드웨어 RAID: 전용 컨트롤러, 더 나은 성능/오프로드
  - 소프트웨어 RAID: 유연성, 저비용(운영체제에서 처리)

## 장애 복구 시나리오
- 디스크 교체 후 rebuild 절차
- 스냅샷과 백업의 중요성(단순 RAID는 백업을 대체하지 않음)

## 권장 실무
- 중요한 데이터는 RAID + 백업 전략 병행
- 모니터링과 정기적인 스크럽(scrub)으로 데이터 무결성 확인

---

### RAID 재구성 예시 (개요)
```
1. 디스크 장애 감지
2. 장애 디스크를 새 디스크로 교체
3. 재구성(rebuild) 시작: 살아남은 디스크를 읽어 패리티를 계산하고 새 디스크에 기록
4. 재구성 속도와 I/O 부하를 모니터링하여 2차 장애를 방지
```

재구성(rebuild) 중에는 degraded mode에서 성능 영향이 클 수 있으므로, 재구성 시간대를 미리 계획하세요.
