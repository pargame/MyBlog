# 파일 시스템 설계

이 문서는 파일 시스템의 기본 구성 요소(메타데이터, 인덱싱, 저널링/로그 구조, 디렉토리 구조)를 정리합니다.

이 문서는 파일 시스템 설계의 핵심 개념과 실무에서 고려해야 할 트레이드오프를 정리합니다.

## 기본 구성 요소
- 메타데이터: inode/extent, 소유자/권한, 타임스탬프
- 데이터 레이아웃: 연속 블록 vs extent 기반 할당
- 디렉토리 인덱싱: 선형 탐색, 해시, B-tree(예: ReiserFS, XFS)

## 일관성 전략: 저널링 vs COW
- 저널링(Journal): 메타데이터(또는 데이터+메타)를 로그에 먼저 기록해 크래시 시 복구 시간을 단축.
- Copy-on-Write(COW): 기존 데이터를 변경하지 않고 새로운 블록을 할당해 쓰는 방식(ZFS, Btrfs). 스냅샷과 데이터 무결성에 유리.

## 동시성과 잠금
- 파일 시스템은 동시성 제어가 중요: fine-grained 락, lock-free 구조, per-inode lock 등을 사용.

## 성능 벤치마크와 설계 고려사항
- IOPS vs throughput: 작은 랜덤 쓰기 workload는 IOPS 민감, 대용량 연속 전송은 throughput 민감.
- 캐싱과 write-back 정책: 데이터 손실 위험과 성능 사이 트레이드오프가 존재.

## 예시: inode vs extent
- inode(전통적 유닉스 FS): 파일의 메타데이터에 직접 블록 포인터를 저장. 큰 파일은 indirect block을 사용.
- extent: 연속 영역을 하나의 범위로 관리해 대형 파일의 메타데이터 오버헤드를 줄임.

## 실습 아이디어
- 간단한 인메모리 파일 시스템을 구현해보고, 저널링과 COW 모드에서의 크래시 복구 동작을 비교해 보세요.
