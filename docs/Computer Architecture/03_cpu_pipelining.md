# 파이프라이닝 심화 (Pipelining Deep Dive)

파이프라인은 명령어 처리 단계를 분리하여 병렬로 실행함으로써 처리량을 향상시킵니다. 이 문서는 파이프라인의 해저드와 해법, 그리고 심화 기법을 다룹니다.

## 파이프라인 단계 예시
1. Fetch
2. Decode
3. Execute
4. Memory
5. Writeback

## 해저드와 해결책
  - 포워딩(Forwarding): 결과를 바로 다음 단계에 전달해 대기 감소
  - 스톨(Stall): 위험이 해소될 때까지 파이프라인을 멈춤
  - 분기 예측(Branch Prediction): 정확한 분기 방향 예측으로 파이프라인 플러시 최소화
  - 지연 슬롯(Delayed Branch): 분기 지침 다음 명령어를 유용하게 사용
  - 자원 복제 또는 파이프라인 재설계

## 추가: 파이프라인 설계의 실무 팁
- 데이터/제어 해저드 식별 후 우선 순위로 해결: forwarding, branch predictor, 혹은 리오더 버퍼
- 파이프라인 깊이 결정은 클럭 주파수와 복잡도(리스크)에 대한 트레이드오프

## 설계 팁
- 파이프라인 깊이와 분기 예측 정확도의 균형 유지
- 복잡한 투기/복구 로직은 검증 비용이 큼 — 설계 단계에서 단순화 가능한지 검토
