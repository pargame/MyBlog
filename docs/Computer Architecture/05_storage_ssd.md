# SSD 내부 구조와 쓰기 증폭

이 문서는 SSD의 플래시 구조, 웨어레벨링, 가비지 컬렉션, 쓰기 증폭(Write Amplification Factor, WAF)에 대해 설명합니다.

이 문서는 SSD의 내부 동작을 개략적으로 설명하고, 성능 및 내구성 설계에서 중요한 개념들을 정리합니다.

## 플래시 기초 (NAND flash)
- 플래시 메모리는 셀(cell)에 전하를 저장해 데이터를 표현. SLC/MLC/TLC/QLC는 셀당 비트 수 차이로 성능·내구성·용량에 영향.
- 최소 쓰기 단위: 페이지(일반적으로 4KB), 최소 지우기 단위: 블록(수십~수백 페이지).

## FTL (Flash Translation Layer)
- FTL은 논리 블록 주소(LBA)와 물리 페이지 주소(PPA)를 매핑. SSD는 인플레이스 업데이트를 지원하지 않으므로 웨어레벨링과 로그 구조가 필수.
- 매핑 전략: 페이지 단위 매핑(full mapping), 세그먼트/블록 단위 매핑, 하이브리드 매핑.

## 가비지 컬렉션(GC)과 쓰기 증폭(WAF)
- GC: 유효 데이터가 적은 블록을 선택해 이동시키고 블록을 지운 뒤 재사용. GC 시 데이터 이동이 발생해 실제 쓰기량이 증가.
- WAF = (총 플래시에 기록된 바이트) / (호스트가 기록한 바이트). 낮을수록 좋음.
- GC 비용을 줄이는 전략: 충분한 오버프로비저닝, 효율적 GC 알고리즘(예: greedy, cost-benefit), 로그 구조 최적화.

## 웨어레벨링과 내구성
- 웨어레벨링은 블록별 쓰기 횟수를 균등화해 수명을 연장. 동작 방식: static vs dynamic wear-leveling.

## 성능/내구성 트레이드오프
- 많은 오버프로비저닝은 WAF를 낮추고 성능을 향상시키지만 용량 효율을 떨어뜨립니다.
- TLC/QLC는 용량은 높지만 프로그램/지우기(P/E) 사이클 수가 적어 내구성이 낮음.

## 예시: 간단한 GC 시나리오
1. 블록 A(페이지 0..127)에 30% 유효 데이터가 있고 블록 B에 90% 유효 데이터가 있을 때, GC는 보통 A를 선택.
2. A의 유효 데이터를 다른 블록으로 복사하고 A를 지운 후 A를 재활용.

## 실습 아이디어
- 소형 시뮬레이터로 LBA->PPA 매핑, 가비지 컬렉션 동작, WAF 측정을 구현해 보세요.
