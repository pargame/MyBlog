# CPU 구조

이 문서는 CPU의 주요 구성요소(레지스터, 인터럽트, 성능 설계, 파이프라이닝 등)와 실제 동작 원리를 다룹니다. 실무에서 자주 마주치는 병목, 설계 포인트, 그리고 구조적 시각화 방법도 함께 제시합니다.

---
## 실무 팁
- CPU 성능은 파이프라이닝, 캐시, 분기 예측 등 미세 설계에 크게 좌우됨
- 인터럽트/예외 처리는 실시간 시스템에서 매우 중요

---
### 연습 문제
1) 파이프라이닝이 없는 CPU와 있는 CPU의 성능 차이를 간단히 설명하라.
2) 인터럽트와 예외의 차이점을 실제 예시와 함께 설명하라.
---


CPU는 연산과 제어의 중심입니다. 이 문서에서는 레지스터, 인터럽트, 성능 향상 기법, 파이프라이닝을 중심으로 설명합니다.

## __레지스터
- 레지스터는 CPU 내부의 소형 고속 저장소로, 연산의 즉시 피연산자와 결과를 저장합니다.
- 일반 목적 레지스터, 특수 목적 레지스터(프로그램 카운터, 상태 레지스터 등)가 있음.
- 레지스터 파일(register file)은 다수의 접근(읽기/쓰기)을 효율적으로 처리하기 위해 설계됨.

## __인터럽트
- 인터럽트는 외부 이벤트나 내부 예외가 발생했을 때 CPU의 현재 흐름을 일시 중단하고 특정 서비스 루틴으로 분기하게 하는 메커니즘.
- 동기적 예외(예: 산술 오버플로우)와 비동기적 인터럽트(예: I/O 완성)가 있음.
- 컨텍스트 스위칭 비용과 우선순위 관리가 중요.

## __CPU 성능 향상을 위한 설계
- 파이프라이닝, 슈퍼스칼라, 명령어 레벨 병렬성(ILP)을 이용
- 분기 예측(branch prediction), 투기 실행(speculative execution)
- 캐시 최적화와 메모리 대역폭 개선

## __파이프라이닝을 통한 명령어 병렬 처리
- 파이프라인: 명령어 실행 단계를 분리(fetch, decode, execute, memory, writeback)하여 서로 다른 명령어가 동시에 동작하도록 함
- 파이프라인 해저드(hazard)
  - 데이터 해저드: 명령어 간 데이터 의존성
  - 제어 해저드: 분기 때문에 발생하는 파이프라인 플러시
  - 구조적 해저드: 하드웨어 자원 충돌
- 해법: 포워딩/데이터 경로 재설계, 분기 예측, 스톨 제어

파이프라이닝은 성능을 크게 높이지만 설계 복잡성과 예측 실패 시 오버헤드를 동반합니다.

### 간단 예제

간단한 명령어 시퀀스(의사어셈블리):
```asm
MOV R1, #5
MOV R2, #7
ADD R3, R1, R2   ; R3 = 12
```

인터럽트 처리기 스켈레톤(자세한 내용은 [[03_cpu_interrupts.md]] 참조):
```c
// pseudo C for ISR
void isr() {
  save_context();
  handle_device();
  restore_context();
}
```
