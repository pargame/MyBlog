# 데이터 사전(Data Dictionary, DD)

시스템에서 사용하는 데이터의 의미, 구조, 형식, 속성 등을 표로 체계적으로 정리한 문서. [[RequirementAnalysis]] 단계에서 요구사항 분석과 데이터 흐름도(DFD) 작성에 필수적으로 활용된다.

---
## 주요 특징
- 데이터의 정의, 속성(타입, 길이, 단위 등), 관계, 제약조건을 명확히 기술
- 데이터 흐름도(DFD)와 연계해 데이터의 흐름과 변환을 추적 가능
- 용어의 혼동 방지, 시스템 간 데이터 표준화에 기여

---
## 실무 활용
- 데이터베이스 설계, 인터페이스 명세, 시스템 통합 시 데이터 불일치 방지
- 신규 개발/유지보수 시 데이터 구조 변경 이력 관리

---
## 예시 (표)
| 데이터 항목 | 설명         | 타입   | 길이 | 제약조건 |
|:-----------:|:------------|:------|:----:|:--------|
| user_id     | 사용자 ID    | int   | 8    | PK, not null |
| name        | 사용자 이름  | char  | 20   | not null |
| email       | 이메일       | char  | 40   | unique |

---
## 연습 문제
1) 데이터 사전이 없는 시스템에서 발생할 수 있는 문제점을 2가지 이상 설명하라.
2) 데이터 사전과 데이터 흐름도(DFD)의 관계를 간단히 서술하라.
