

> 사용자의 요구를 체계적으로 수집·정리·분석해 개발에 반영할 수 있도록 명확히 정의하는 과정. 분석이 부실하면 개발 방향 혼란, 오류·재작업 발생.

---
**주요 단계**
- 수집: 인터뷰, 설문, 관찰 등으로 요구 파악
- 분류: 기능/비기능, 우선순위 등으로 정리
- 검토: 충돌·모호함·누락 등 문제점 확인 및 해결
- 합의: 이해관계자와 협의해 요구사항 확정

**주요 도구/기법**
- [[DFD]]: 데이터 흐름도
- [[DD]]: 데이터 사전
- [[CASE]]: 소프트웨어 공학 자동화 도구
- [[HIPO]]: 계층적 입력·처리·출력 다이어그램
- [[UML]]: 표준 객체지향 모델링 언어

**관련 문서**: [[Requirement]]
