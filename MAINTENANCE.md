## Maintenance

이 문서는 간단 요약본입니다. 전체 운영 규칙과 체크리스트는 `handbook/MAINTENANCE.md`를 참고하세요.

- 배포: `main` 브랜치로 푸시하면 GitHub Pages가 자동 배포합니다.
- 작업 순서: 변경 → 커밋 → 푸시. 작은 단위로, 메시지는 명확하게.
- 로컬 확인(선택): `npm run build` 실행 후 산출물(`public/graph.json`, `public/meta.json`) 점검.

언어 정책 요약:
- 설명/채팅은 한국어, 개발 관련(코드/식별자/커밋 메시지/주석)은 영어, 사용자 문서는 한국어.
